# 📘 프로젝트 기획서

## 프로젝트명

Personal Finance APP

---

## 1. 개요

본 시스템은 개인의 소비·수입·투자 데이터를 통합 관리하고,
자동 분석 및 주간/월간 리포트를 통해 자산 증식을 지원하는
DB 중심 재무 운영 시스템이다.

목표는 단순 가계부가 아니라
**현금흐름 통제 + 투자 비율 관리 + 순자산 증가 추적**이다.

---

## 2. 목표

### 🎯 핵심 목표

* 월간/주간 소비 패턴 자동 분석
* 순자산(Net Worth) 자동 계산
* 저축률 및 투자 비율 추적
* 소비 급증 및 예산 초과 경고
* Discord 자동 리포트 전송

### 📈 장기 목표

* 자산 배분 분석
* 복리 성장 추적
* 위험 소비 탐지
* 정기결제(구독) 자동 탐지

---

## 3. 시스템 아키텍처

```
[거래 데이터(CSV 등)]
        ↓
      n8n (ETL)
        ↓
   PostgreSQL (Core DB)
        ↓
Metabase (분석/대시보드)
        ↓
Discord (자동 리포트)
```

---

## 4. 기술 스택

| 구성요소   | 기술                         |
| ------ | -------------------------- |
| 자동화    | n8n                        |
| 데이터베이스 | PostgreSQL                 |
| 분석/BI  | Metabase                   |
| 알림     | Discord Webhook            |
| 배포환경   | Home Server (Docker 기반 권장) |

---

## 5. 데이터 설계 개요

### 핵심 테이블

**accounts**

* 계좌 유형 (일반/카드/증권/현금)
* 현재 잔액

**transactions**

* 날짜
* 금액
* 유형 (income / expense / transfer)
* 계좌
* 가맹점
* 메모

**categories**

* 카테고리 트리 구조

**transaction_category**

* 거래-카테고리 매핑 (N:N 구조)

**investments**

* 종목
* 수량
* 평균단가
* 평가금액

---

## 6. 주요 기능

### 1️⃣ 데이터 수집

* CSV 기반 거래 데이터 적재
* 중복 방지 Upsert 처리

### 2️⃣ 자동 분류

* 가맹점/메모 기반 규칙 분류

### 3️⃣ 주간 리포트

* 총 지출
* 카테고리 Top 3
* 전주 대비 증감
* 경고 항목

### 4️⃣ 월간 리포트

* 순자산 증감
* 저축률
* 투자 비율
* 자산 배분 현황

---

## 7. 핵심 지표 (KPI)

* 월 저축률 ≥ 30%
* 투자 비율 ≥ 40%
* 고정비 비율 ≤ 50%
* 순자산 월간 증가 유지

---

## 8. 구현 단계

### 1단계

* DB 스키마 설계
* n8n CSV 적재 플로우 구축

### 2단계

* 자동 분류 로직 구축
* 주간/월간 집계 SQL 작성

### 3단계

* Discord 리포트 자동화
* Metabase 대시보드 구성

---

## 9. 기대 효과

* 소비 통제 시스템 구축
* 투자 비율 자동 관리
* 자산 증가 구조 가시화
* 재무 의사결정 자동 지원
