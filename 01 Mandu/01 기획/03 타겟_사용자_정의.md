
1.3 주요 타깃 사용자

 1) PM / 기획자 (1순위)

* 역할
  * 프로젝트 계획 수립
  * Flow 설계
  * Task 분해 / 우선순위 / 마감 관리
  * 프로젝트 진행 상황 모니터링, 리스크 관리
* 이 서비스에서 하고 싶은 일
  * 설계한 Flow Chart 기준으로, 각 단계별 Task/작업 진행 상태를 확인
  * 마감이 다가오는데 작업이 거의 없는 구간을 빠르게 찾기
  * 프로젝트별/기간별 활동 지표와 평가 점수 확인
  * 회의/보고에 사용할 수 있는 주간·월간 요약 관점 확보


2) 팀 리더 / 리드 개발자 (2순위)

* 역할
  * 팀의 개발 일정·우선순위 조정
  * 병목·리스크가 있는 Task를 조기 발견
* 이 서비스에서 하고 싶은 일
  * 특정 Flow/기능의 구현 진행 상황을 Task·Commit 기준으로 확인
  * 담당자별 작업 분포/부하 확인

3) 개발자 (3순위)

* 역할
  * Task를 처리하고 Git에 커밋을 남김
* 이 서비스에서 하는 일
  * 자신에게 할당된 Task 확인
  * 커밋 메시지에 Task 번호를 포함해 Task–Commit이 자동으로 연결되게 사용
  * Flow/Task 단위로 자신이 어떤 부분을 담당하고 있는지 확인
