# 서버 악성감염
## 개요
- 최근 React 생태계에서 발견된 고위험군 취약점인 CVE-2025-55182를 이용한 원격 코드 실행(RCE) 공격으로 서버가 악성 감염 확인

## 취약점 분석: CVE-2025-55182
해당 취약점은 React 프레임워크의 특정 렌더링 로직에서 발생하는 보안 허점으로, 공격자가 악의적인 스크립트를 주입하여 서버 권한을 획득할 수 있는 치명적인 결함입니다.

- 위험도: Critical (NVD 기준)
- 공격 벡터: SSR(Server Side Rendering) 환경에서 클라이언트 입력값이 충분히 검증되지 않을 경우 발생.
- 영향: 시스템 제어권 탈취, 내부 데이터 유출, 봇넷 가담 등.

## 대응 및 복구
### Phase 1: 긴급 격리 및 초기화
- 네트워크 연결 차단 및 중요 설정 파일 백업 (악성코드 포함 여부 정밀 검사).
- OS 레벨의 오염 가능성을 배제하기 위해 스토리지를 완전히 포맷 후 Ubuntu 24.04 LTS 재설치.
### Phase 2: 인프라 재구축 및 강화
단순 복구가 아닌, 다시는 이런 일이 발생하지 않도록 보안 강화와 자동화에 초점을 맞췄습니다.  
- Nginx 역방향 프록시 및 SSL 적용: Cloudflare와 연동하여 Full (Strict) SSL 모드 적용 및 암호화되지 않은 접근 차단.
